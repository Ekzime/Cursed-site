# Cursed Board

Хоррор-проект: проклятый форум с элементами ритуала.

## Концепт

Старый заброшенный форум/имиджборд. На первый взгляд — обычный архив мёртвого сообщества. Но чем больше пользователь взаимодействует, тем страннее становится контент.

### Механика "Проклятого Форума"

- Посты появляются "сами" — новые сообщения в старых тредах
- Юзеры с аномальными датами регистрации (из будущего, до создания интернета)
- Картинки/контент "портятся" с каждым просмотром
- Одинаковые сообщения от разных пользователей в разное время
- Посты которые "знают" что ты их читаешь
- Удалённые посты которые возвращаются
- Треды про события которые ещё не произошли

### Механика "Ритуала"

- Каждый запрос к API — часть невидимого ритуала
- Сервер отслеживает "прогресс" пользователя
- Контент адаптируется под историю взаимодействий
- Определённые последовательности действий открывают скрытый контент
- В какой-то момент форум начинает "узнавать" пользователя
- Персонализированные аномалии на основе поведения

### Процедурная генерация

- Посты, юзернеймы, даты
- Аномалии и их паттерны
- Связи между постами и событиями
- "Глубина" проклятия зависит от активности

## Технологии

- **Backend**: FastAPI + Celery + Redis
- **Database**: MySQL
- **Frontend**: React + Vite
- **Realtime**: WebSocket (FastAPI native)
- Процедурная генерация контента

См. `docs/architecture-v1.md` для полной архитектуры.

## Атмосфера

Киберпанк/современность, аномалии, крипипаста-вайб. Не джампскеры — медленное нарастающее ощущение неправильности.
